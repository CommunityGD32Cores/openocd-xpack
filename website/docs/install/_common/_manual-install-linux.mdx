### Download

The GNU/Linux versions of **xPack { siteConfig.customFields.appName }**
are packed as `.tar.gz` archives.
Download the latest version named like:

* <code>xpack-{ siteConfig.customFields.appLcName }-{ siteConfig.customFields.upstreamVersion }-{ siteConfig.customFields.xpackSubversion }-linux-x64.tar.gz</code>
* <code>xpack-{ siteConfig.customFields.appLcName }-{ siteConfig.customFields.upstreamVersion }-{ siteConfig.customFields.xpackSubversion }-linux-arm.tar.gz</code>
* <code>xpack-{ siteConfig.customFields.appLcName }-{ siteConfig.customFields.upstreamVersion }-{ siteConfig.customFields.xpackSubversion }-linux-arm64.tar.gz</code>

:::note

In case you wonder where the suffix comes
from, it is exactly the Node.js `process.platform` and `process.arch`.

:::

### Unpack

To manually install the <b>xPack { siteConfig.customFields.appName }</b>,
unpack the archive and move it
to <code>~/.local/xPacks/{ siteConfig.customFields.appLcName }/xpack-{ siteConfig.customFields.appLcName }-{ siteConfig.customFields.upstreamVersion }-{ siteConfig.customFields.xpackSubversion }</code>:

<CodeBlock language="console"> {
`mkdir -p ~/.local/xPacks/${ siteConfig.customFields.appLcName }
cd ~/.local/xPacks/${ siteConfig.customFields.appLcName }

tar xvf ~/Downloads/xpack-${ siteConfig.customFields.appLcName }-${ siteConfig.customFields.upstreamVersion }-${ siteConfig.customFields.xpackSubversion }-linux-x64.tar.gz
chmod -R -w xpack-${ siteConfig.customFields.appLcName }-${ siteConfig.customFields.upstreamVersion }-${ siteConfig.customFields.xpackSubversion }`
}</CodeBlock>
